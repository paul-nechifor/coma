Test Plan for: ComaA Adding a new route

Persona Type Tested: Primary

Duration: 1–2 minutes

Complete the following before starting: Greeting and Consent & User Background Information.

Overall Objectives
  • Assess the overall effectiveness of the ComA for different types of users performing basic, common tasks;
  • Identify obstacles to completing a certain task or using a certain component;
  • Assess if the icons used are understood;
  • Evaluate if enough time was provided to do the task;
  • Assess if the responsiveness of the application is sufficient for the user.
Scenario
  1. The user wants to add a new route
  2. He opens the application and selects the routes tab
  3. He taps the “Add new route” button
  4. He figures out that tapping the maps brings up the location selection activity
  5. He changes both locations and selects the arrival time
  6. He moves to the “Transports” tab and gets the list of means of transport that he needs to take
  7. He confirms the route by taping “Done”
Tasks
  1. selecting the routes tab: 10 seconds
  2. opening the route adding activity: 10 seconds
  3. inputing the start and end locations: 1 minute
  4. inputing the arrival time: 15 seconds
  5. reviewing the list of means of transport: 30 seconds
  6. confirming: 5 seconds
Post-test Questions
  1. Was it hard to figure out how to input the two locations?
  2. Did you try to change the orientation in any of the fragments?
  3. Did you find the translation to human (i.e street name) address usefull?
  4. Did you find it hard to figure out that there are two different views for the checked/unchecked state of the box?
  5. Did you try to click Done without moving to the Transports tab? Did you find that it was correct to not close the activity, but instead to move to that tab?
Notes for Test Coordinator
  • Remind the user that you would like them to 'think aloud' so that you can understand what they are thinking, about the task or the product/application;
  • Read the user the scenario for the test;
  • Read the user the tasks he has to complete. If the tasks are complex, you may want to put a sheet of paper in front of the user with each task written out;
  • Record the user's actions, comments, questions, and body language (write them in the Test Observations section accompanied by video/audio).
  • Don't offer help; let the user attempt to perform the task. If they ask for help reply with: "What do you think you/that would do?" "What do you think that means?".
