Ideas

1. Android Application

The mobile aplication will be developed for the Android platform.

2. Associated Web Site

The website will be used for downloading updated information about the means of transport and points of interest. The user (if he chooses to) will send data about his location when in transit in order to describe the current state of public transport and provide others with information about the tardiness of the means of transport and related data.

3. Supplimental information

The user app can also receive information from other services:

  • weather information from wunderground.com for informing the user about what he should expect when leaving his house and when he returns;
  • Facebook for getting the events to which he is invited to and responding to the discution if he is late;
  • Geocoder for getting the human address for specific coordinates.

Design Patterns

  • Vertical Navigation Bar (app menu)
  • Account Registration
  • Lazy Registration (phone info is used if an account is not registered)
  • Forgiving Format (in searching for a location)
  • Good Defaults (when creating a route, the sleep interval)
  • Home Link (in the various views)
  • Event Calendar (in the next actions)
  • Scanning Labels (on registration, login, route editing
